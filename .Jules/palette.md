## 2024-04-10 - Add ARIA labels and focus outlines to dynamically generated interactive elements
**Learning:** In the S.I.L.K. web dashboard, icon-only buttons created dynamically using `document.createElement` in `static/js/dashboard.js` must have `aria-label` applied explicitly via `setAttribute` since they lack textual content. Also, missing focus outlines for keyboard navigation was noted across both static template buttons and dynamically generated JS components. This dashboard relies on Tailwind utilities `focus:outline-none focus:ring-2 focus:ring-[#EBB2FF] rounded` to maintain consistent Cyber-Industrial focus visuals without relying on default browser focus rings.
**Action:** Whenever creating icon-only buttons or interactive elements (especially dynamically in JS), explicitly invoke `el.setAttribute("aria-label", "Meaningful action");` and append `focus:outline-none focus:ring-2` Tailwind classes to ensure they are accessible to screen readers and keyboard navigators.
